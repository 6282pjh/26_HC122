# 💡 1. 프로젝트 개요

## 1-1. 프로젝트 소개

- **프로젝트 명** : 월드 모델 기반 행동 시뮬레이션을 제공하는 수술 어시스턴트 로봇
- **프로젝트 정의** : 집도의의 명령과 피드백을 반영하여, 수술 보조 동작을 실제 수행 전에 예측·시뮬레이션하고 승인 후 실행하는 Human-in-the-loop 기반 협업형 수술 보조 시스템

![프로젝트 대표 이미지](이미지_추가예정)

## 1-2. 개발 배경 및 필요성

- 현대 수술 환경은 환자별 해부학적 구조와 조직 상태가 모두 달라 높은 정밀성과 실시간 판단 능력을 요구합니다.
- 기존 수술 로봇은 정밀한 조작에는 강점이 있으나, 집도의와의 유기적인 협업이나 수술 전 행동 검증 구조는 충분하지 않은 한계가 있습니다.
- 특히 절개, 봉합, 도구 전달과 같은 반복적이고 정밀한 수술 보조 작업은 의료진의 피로도와 업무 부담을 증가시키며, 실수 가능성도 함께 높아집니다.
- 따라서 본 프로젝트는 로봇이 동작을 바로 수행하는 것이 아니라, **행동을 먼저 시뮬레이션하여 집도의가 검토·수정·승인한 뒤 실행하는 구조**를 통해 더 안전하고 신뢰성 있는 협업형 수술 보조 시스템을 구현하고자 합니다.

## 1-3. 프로젝트 특장점

- 집도의의 최종 판단권을 보장하는 **Human-in-the-loop 협업 구조**
- NVIDIA Cosmos 기반 **행동 예측 시뮬레이션**을 통한 사전 검증
- 디지털 트윈 기반 검증 환경과 실제 로봇을 연결하는 **Sim-to-Real 확장성**
- 3D 비전 기반 공간 인식과 Jetson 기반 제어를 결합한 **정밀 수술 보조 시스템**
- 단순 자동화가 아닌 **도구 전달, 절개 보조, 봉합 보조** 중심의 현실적 수술 어시스턴트 구조
- 수술 보조뿐 아니라 의료 훈련, 정밀 작업, 바이오 실험 자동화 등으로의 **확장 가능성**

## 1-4. 주요 기능

- **수술 환경 및 도구 인식** : 3D 비전 카메라를 통해 수술 공간과 도구의 위치를 인식하고 정밀 작업에 필요한 환경 정보를 파악
- **월드 모델 기반 미래 행동 예측 시뮬레이션** : 절개, 봉합, 도구 전달 동작을 실제 수행 전에 가상환경에서 시뮬레이션
- **피드백 기반 행동 수정** : 집도의가 절개 길이·깊이·위치·방향, 도구 전달 순서, 봉합 범위 등을 수정하면 이를 반영해 행동 계획 재조정
- **정밀 수술 보조 수행** : 승인된 계획에 한해 mm 단위 절개 보조, 도구 전달, 봉합 보조 수행
- **디지털 트윈 기반 검증 시스템** : 실제 로봇과 연동된 가상환경에서 동작을 사전 검증하고 안전성 및 신뢰성 향상
- **집도의 통합 인터페이스** : 대시보드와 음성 입력을 통해 로봇 행동 승인·수정·재계산 수행

## 1-5. 기대 효과 및 활용 분야

- **기대 효과**
  - 수술 보조 동작을 사전에 예측·검증함으로써 수술 중 불확실성과 위험 요소를 감소
  - 반복적이고 정밀한 보조 업무를 로봇이 분담하여 의료진의 피로도와 업무 부담 완화
  - 작업의 정밀성, 일관성, 효율성 향상
  - 디지털 트윈과 행동 시뮬레이션을 활용한 안전 중심 의료 보조 기술 확보

- **활용 분야**
  - 수술실 내 도구 전달, 절개 보조, 봉합 보조
  - 의료 훈련 및 수술 시뮬레이션
  - 디지털 트윈 기반 검증 시스템
  - 바이오 실험 자동화
  - 정밀 검사·정밀 조작 공정 등 협업형 정밀 작업 분야

## 1-6. 기술 스택

- **프론트엔드 / UI** : Flutter, Figma
- **백엔드 / 서버** : Python, AWS, MySQL
- **AI / 시뮬레이션** : NVIDIA Cosmos, NVIDIA Isaac Lab, Reinforcement Learning, Sim-to-Real
- **로봇 제어** : ROS2 Humble, MoveIt2, RViz, Python
- **임베디드 / 하드웨어** : NVIDIA Jetson, 3D Vision Camera, Depth Camera, Motor Control, 3D Printing
- **데이터 처리** : ROSBag, JSON Dataset, 행동 데이터 로그 관리
- **협업 / 관리** : GitHub, Notion
